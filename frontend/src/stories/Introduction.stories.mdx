# PZR4 Design System

Welcome to the PZR4 Design System Storybook! This is a living documentation of all UI components used in the Padel Club Management System.

## 🎨 Design Principles

Our design system is built on these core principles:

1. **Consistency** - Unified experience across all components
2. **Accessibility** - WCAG AA compliant components
3. **Performance** - Optimized for speed and efficiency
4. **Flexibility** - Adaptable to different use cases
5. **Modern** - Clean, contemporary design

## 🏗️ Architecture

### Technology Stack
- **Framework**: Next.js 14 with App Router
- **Styling**: Tailwind CSS with CSS Variables
- **Components**: Radix UI primitives
- **Animations**: Framer Motion
- **Icons**: Lucide React

### Design Tokens
- **Colors**: Semantic color system with dark mode support
- **Typography**: System font stack for optimal performance
- **Spacing**: 4px base unit system
- **Shadows**: Apple-inspired elevation system

## 📚 Component Categories

### Foundation
- **Colors** - Color palette and usage
- **Typography** - Text styles and hierarchy
- **Spacing** - Spacing system and guidelines
- **Icons** - Icon library and usage

### Components
- **UI Components** - Buttons, Inputs, Cards, etc.
- **Layout** - Grid, Container, Spacer
- **Feedback** - Alerts, Toasts, Loading states
- **Data Display** - Tables, Lists, Stats

### Patterns
- **Forms** - Form layouts and validation
- **Navigation** - Headers, Sidebars, Breadcrumbs
- **Empty States** - No data scenarios
- **Error Handling** - Error states and recovery

## 🚀 Getting Started

### For Developers
```bash
# Install dependencies
npm install

# Run Storybook
npm run storybook

# Build Storybook
npm run build-storybook
```

### For Designers
- Browse components in the sidebar
- Play with controls to see variations
- Copy code snippets for implementation
- Check accessibility information

## 📖 How to Use This Storybook

1. **Browse Components** - Use the sidebar to navigate
2. **Interact with Controls** - Modify props in real-time
3. **View Code** - Click "Show code" to see implementation
4. **Check Docs** - Read component documentation
5. **Test Accessibility** - Use the accessibility tab

## 🔄 Contributing

When adding new components:

1. Create a `.stories.tsx` file
2. Document all props and variations
3. Include usage examples
4. Add accessibility notes
5. Test in light and dark modes
