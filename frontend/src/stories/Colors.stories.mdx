## Usage Guidelines

### CSS Variables
```css
/* Light theme */
:root {
  --primary: 222.2 47.4% 45.2%;
  --primary-foreground: 210 40% 98%;
  --secondary: 210 40% 96.1%;
  --secondary-foreground: 222.2 47.4% 11.2%;
  /* ... more variables */
}

/* Dark theme */
.dark {
  --primary: 210 40% 98%;
  --primary-foreground: 222.2 47.4% 11.2%;
  --secondary: 217.2 32.6% 17.5%;
  --secondary-foreground: 210 40% 98%;
  /* ... more variables */
}
```

### Usage in Components
```tsx
// Using with Tailwind
<div className="bg-primary text-primary-foreground">
  Primary colored element
</div>

// Using semantic colors
<Button variant="destructive">Delete</Button>
<Badge variant="success">Active</Badge>
<Alert variant="warning">Warning message</Alert>
```

### Best Practices

1. **Use semantic colors** - Prefer `primary`, `secondary`, etc. over specific color values
2. **Maintain contrast** - Ensure text is readable on all backgrounds (WCAG AA)
3. **Consistent usage** - Use the same color for the same purpose throughout the app
4. **Dark mode support** - Always test components in both light and dark themes
5. **Accessibility** - Check color contrast ratios for all text/background combinations

### Color Application Examples

- **Primary**: Main CTAs, active states, links
- **Secondary**: Secondary buttons, subtle backgrounds
- **Destructive**: Delete actions, error states
- **Success**: Confirmations, positive feedback
- **Warning**: Cautions, important notices
- **Muted**: Disabled states, placeholder text
